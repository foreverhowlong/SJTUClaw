# 课程报告

## 任务契约

开始前提取并整理：报告类型、课程或实验名称、主题、核心问题、篇幅、格式要求、参考材料、截止要求和目标路径。

- 缺少会显著改变结构或结论的关键条件时，只追问必要问题。
- 缺少次要信息时采用保守假设，在交付中明确说明，不让可合理补全的细节阻塞任务。
- 用户给出的课程要求优先于本 Skill、模板和一般写作惯例。

## 材料与证据

1. 需要 workspace 材料时，先用 `list_dir`、`read_file` 或附件工具读取实际内容，再开始写作。
2. 建立简短的证据清单：材料来源、可支持的观点、关键概念或数据，以及材料不能支持什么。
3. 明确区分材料事实、用户观点和综合分析。无法核实的信息必须标注为待确认，不得编造引文、页码、数据、实验现象或参考文献。

## 写作流程

1. 先用一句话确定中心问题，再给出能够直接回答该问题的中心论点或报告目标。
2. 从 `assets/template.md` 选择合适结构。模板是起点，不是必须逐项照搬的固定目录。
3. 先列出章节目的和论证顺序，再展开正文。分析类段落尽量形成“观点—证据—解释—与主题的关系”；实验类段落应区分方法、结果和解释。
4. 摘要应准确概括问题、方法或材料、主要结论，不写成背景介绍；结论应回应开头提出的问题，不只重复正文标题。
5. 控制篇幅和信息密度：优先删去重复表达，不用空泛套话凑字数。

## 交付

1. 交付前逐项执行 `references/checklist.md`；发现证据不足时应缩小结论或明确标注，而不是补写虚构内容。
2. 用户要求保存时，使用 `create_file`、`overwrite_file` 或 `edit_file` 写入 workspace，并经过现有 approval。工具成功前不得声称文件已经保存。
3. 最终答复简要说明报告结构、实际保存路径、采用的关键假设，以及仍需用户确认或补充的内容。
