# 交付检查清单

## 要求与结构

- 报告类型、主题、篇幅和格式符合用户或课程要求。
- 标题准确，章节顺序能回答核心问题，Markdown 标题层级连续。
- 摘要包含问题、材料或方法和主要结论；结论回应开头，而不是机械复述。

## 证据与分析

- 关键观点能追溯到实际材料，事实、用户观点和综合分析没有混写。
- 引文、页码、数据、实验现象和参考文献均未虚构；无法核实之处已明确标注。
- 每个主要章节都有具体信息或分析，不靠重复表达、空泛评价和模板化套话凑篇幅。
- 实验报告已区分客观结果与原因解释，并说明异常、误差或方法限制。

## 交付

- 专有名词、数字、文件名和保存路径前后一致。
- Markdown 可直接阅读，没有残留占位符、写作提示或未闭合的代码块。
- 如果要求保存，目标位于 workspace 内，且只有在写入工具成功后才报告实际路径。
- 最终答复列出关键假设、未解决问题和需要用户补充的材料。
