# 材料汇总

## 任务契约

先确定材料范围、汇总目的、目标读者、期望粒度和交付形式。用户没有指定结构时，根据任务选择“快速简报”“主题综述”“学习笔记”或“行动清单”，并说明选择。

## 读取与记录

1. 使用 `list_dir`、`read_file` 或 `read_attachment` 读取实际内容。先列出纳入范围；缺失、重复、无法读取或疑似过期的材料要单独说明。
2. 先逐来源提取，再跨来源综合。对每份材料记录：来源名称、核心主张、关键事实或例子、限制条件，以及与任务的关系。
3. 尽可能使用文件名和明确的小节名称保留来源线索。没有可靠定位信息时，不编造页码、段落号或出处。

## 综合方法

1. 按问题或主题聚类，而不是按文件顺序简单拼接摘要。
2. 区分共识、互补信息和真实分歧；不得把不同作者、不同时间或不同语境下的观点合并成一个无来源结论。
3. 将事实、原文主张和你的推断分开表达。推断使用“综合判断”或“推断”标记，并说明依据。
4. 合并重复信息，保留能够改变结论的例外、条件和反例。

## 输出与交付

默认按以下顺序组织，可依据用户目标删减：

1. **一句话概览**：材料整体回答了什么问题。
2. **核心发现**：按主题列出结论及对应来源。
3. **分歧与限制**：冲突观点、证据缺口和适用边界。
4. **来源地图**：每份材料的作用，而不是重复完整摘要。
5. **待确认事项与下一步**：需要补充的材料、问题或行动。

用户要求保存时，使用 `create_file`、`overwrite_file` 或 `edit_file` 写入 Markdown 文件，并经过现有 approval；工具成功前不得声称已经保存。最终答复说明纳入了哪些材料、遗漏了哪些材料以及实际保存路径。
